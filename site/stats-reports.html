Extensive Statistical Test Results

This section is still in progress, as doing extensive statistical testing on the RNG requires large amounts of data, and also some choices about which external source code to use. At first I tried to use NIST's 800-22 test suite, but ran into some flaky tests that failed even on well-known cryptographically secure PRNGs (like the Overlapping tempalte test). Apparently this is a known bug. So I pivoted towards using this improved implementation of the NIST test suite. It does not give as nice of a results breakdown as the original NIST test suite, but it does give succinct summaries about the results (number of tests passed, how many of each category failed). Since a non-zero amount of tests are expected to fail for random data, I decided to run the tests on a state-of-the-art CSPRNG alongside my generator, so we have a baseline to compare against. I simply used the python "secrets" module, which uses the OS's CSPRNG, which in my case is Linux's /dev/random, which uses ChaCha20. Below is a preliminary test run: first, 10 separate tests of 100 bit streams (the default bitstream length of 2^20 was used for all tests), and then four tests of 800 bitstreams. As you can see, in both cases there is no statistical significance in the difference of the test results; both the Goobits generator and the CSPRNG fail the tests at about the same rate, which we can conclude is around the expected rate for random data. Around 400MB was used for this preliminary test.
